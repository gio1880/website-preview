File Management Best Practices

Do's
  • Use clear, descriptive file names
  • Include version numbers (v1.0, v2.1)
  • Maintain organized folder structure
  • Update change logs regularly
  • Archive old files instead of deleting
Don'ts
  • Use vague names like "untitled" or "new"
  • Save files in wrong folders
  • Overwrite important files without backup
  • Forget to document major changes
  • Delete files without team approval
Team Members

Full edit access to all folders and files for active collaboration

Mentors

View and comment access to provide guidance and feedback

Judges

View-only access to specific presentation and documentation files
